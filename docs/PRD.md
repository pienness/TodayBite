# TodayBite — 智能外卖推荐助手 产品需求文档 (PRD)

> 版本: v0.1 | 日期: 2026-03-24 | 状态: 方案评审中

---

## 1. 产品概述

### 1.1 一句话定位
基于用户位置获取附近外卖信息，结合预算、热量、营养素、口味偏好，智能推荐最优外卖组合。

### 1.2 目标用户
- 关注饮食健康、有预算限制的外卖用户
- 患有"今天吃什么"选择困难症的上班族/学生群体
- 有控制热量、营养均衡需求的健身/减脂人群

### 1.3 核心价值
| 痛点 | 解决方案 |
|---|---|
| 不知道吃什么 | 智能推荐，减少决策疲劳 |
| 想控制预算 | 精确预算范围筛选（±3元） |
| 想控制热量 | LLM + 食物数据库估算热量 |
| 有饮食禁忌 | 口味/食材排除标签系统 |
| 想营养均衡 | 营养素偏好标签匹配 |

## 2. 功能需求

### 2.1 模块一：外卖数据层

#### 2.1.1 数据结构

**店铺 (Shop)**
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 店铺唯一标识 |
| name | string | 店铺名称 |
| rating | number | 综合评分 (1-5) |
| distance | number | 距离 (米) |
| delivery_fee | number | 配送费 (元) |
| min_order | number | 起送价 (元) |
| delivery_time | string | 预计配送时间 |
| tags | string[] | 店铺分类标签 (川菜/粤菜/快餐...) |

**商品 (Product)**
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 商品唯一标识 |
| shop_id | string | 所属店铺 |
| name | string | 菜品名称 |
| price | number | 价格 (元) |
| description | string | 菜品描述 |
| image_url | string | 商品图片 |
| category | string | 分类 (主食/小吃/饮料/汤品) |
| monthly_sales | number | 月销量 |

**评论 (Review)**
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 评论唯一标识 |
| product_id | string | 关联商品 |
| shop_id | string | 关联店铺 |
| rating | number | 评分 (1-5) |
| content | string | 评论文本 |
| tags | string[] | 评论标签 (份量大/味道好/包装好...) |

#### 2.1.2 MVP 阶段数据策略
- **P0 阶段**：使用精心构造的 Mock 数据（覆盖 15-20 家店铺，每店 10-20 个菜品）
- **P1 阶段**：调研真实数据接入方案（平台合作/用户导入/其他）
- Mock 数据需覆盖多种菜系：川菜、粤菜、日料、轻食沙拉、快餐、面食等

### 2.2 模块二：用户偏好输入

#### 2.2.1 餐标预算
- 输入方式：滑块 + 数字输入框
- 范围：5 - 100 元
- 系统在 **目标价格 ± 3 元** 范围内筛选
- 注意：总价 = 菜品总价 + 配送费（多店时叠加配送费）

#### 2.2.2 热量控制
- 输入方式：预设档位 + 自定义输入
- 预设档位：
  - 轻食 (~400 kcal)
  - 正常 (~600 kcal)
  - 大餐 (~800 kcal)
  - 自定义 (手动输入)
- 界面需标注：**"热量为 AI 估算值，仅供参考"**

#### 2.2.3 营养素偏好 (多选 Tag)
正向选择（希望包含）：
- 高蛋白
- 膳食纤维
- 维生素 B 族
- 维生素 C
- 低脂
- 低碳水
- 富含铁
- 富含钙
- Omega-3

#### 2.2.4 口味偏好

**正向偏好**（多选）：
- 辣
- 微辣
- 清淡
- 酸甜
- 咸鲜
- 麻辣

**排除项**（多选，硬约束）：
- 不吃辣
- 不吃蒜
- 不吃香菜
- 不吃猪肉
- 不吃牛肉
- 不吃海鲜
- 不吃内脏
- 素食
- 无乳糖
- 无麸质

### 2.3 模块三：智能推荐引擎

#### 2.3.1 推荐流程

```
用户输入偏好
    │
    ▼
获取附近店铺 & 菜品数据
    │
    ▼
【硬筛选】── 排除口味禁忌菜品（含关键食材的菜品）
    │         排除明显超出预算的菜品
    │
    ▼
【LLM 分析】── 批量分析候选菜品：
    │           · 热量估算（菜名 + 描述 + 评论 → 估算 kcal）
    │           · 主要营养素推断
    │           · 口味特征提取
    │           · 食材识别（用于排除项二次校验）
    │
    ▼
【打分排序】── 多维度加权评分：
    │           · 热量匹配度 (权重 30%)
    │           · 营养素匹配度 (权重 20%)
    │           · 价格匹配度 (权重 20%)
    │           · 店铺评分 (权重 15%)
    │           · 配送距离/时间 (权重 15%)
    │
    ▼
【组合优化】── 优先级：单店方案 > 双店 > 三店（最多3家）
    │           约束条件：
    │           · 总价（含配送费）在预算 ± 3 元内
    │           · 总热量接近用户目标
    │           · 每家店满足起送价
    │
    ▼
输出 Top 3 推荐方案（附置信度评分）
```

#### 2.3.2 推荐输出内容

每个推荐方案包含：
| 输出项 | 说明 |
|---|---|
| 推荐菜品列表 | 菜名、价格、图片、所属店铺 |
| 预估总热量 | AI 估算，标注 "约 XXX kcal (估算)" |
| 预估总价 | 菜品价 + 各店配送费 |
| 营养素概览 | 主要营养素雷达图/标签 |
| 综合评论摘要 | LLM 对相关评论的正面/负面摘要 |
| 用餐建议 | 如 "此餐蛋白质充足但蔬菜偏少，建议搭配水果" |
| 方案评分 | 综合匹配度百分比 |

#### 2.3.3 交互功能
- **换一批**：重新生成推荐（排除已展示的方案）
- **不喜欢某道菜**：将该菜品加入本次排除列表，重新推荐
- **调整偏好**：快速返回修改偏好参数

### 2.4 模块四：辅助功能

#### 2.4.1 时段感知
- 根据当前时间自动识别用餐场景
  - 06:00-09:30 → 早餐模式
  - 11:00-13:30 → 午餐模式
  - 14:00-17:00 → 下午茶模式
  - 17:30-20:00 → 晚餐模式
  - 20:00-02:00 → 夜宵模式
- 不同时段调整默认预算和热量推荐值

#### 2.4.2 偏好记忆 (MVP 阶段)
- 使用 localStorage 存储用户上次的偏好设置
- 下次打开自动填充，用户可修改

#### 2.4.3 推荐历史
- 本地存储最近 20 条推荐记录
- 可查看历史推荐详情

## 3. 非功能需求

| 维度 | 要求 |
|---|---|
| 性能 | 推荐结果 3 秒内返回（LLM 部分可流式输出） |
| 兼容性 | 支持 Chrome/Safari/Firefox 最新两个大版本，移动端适配 |
| 安全性 | API Key 不暴露在前端，通过后端代理调用 LLM |
| 可用性 | LLM 服务不可用时，降级为纯规则推荐（无热量估算） |

## 4. 页面结构

```

├── 定位显示 & 确认
├── 偏好设置面板
│   ├── 预算滑块

│   └── 用餐建议

├── "调整偏好" 按钮
│
推荐历史页 (/history)
└── 历史推荐列表
```
